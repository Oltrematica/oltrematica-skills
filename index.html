Install

Two commands.

This is the path that installs the verification gate along with the skills. Requires Claude Code.

$ claude plugin marketplace add https://github.com/Oltrematica/oltrematica-skills.git
$ claude plugin install oltrematica-skills@oltrematica

Then, in any repository: say "audit our harness" and read what comes back. There is a scripts/install.sh fallback for repos that can't use the marketplace — it ships the skills but not the gate.

Why

Three failures. None of them announces itself.

The harness fails through invisibility

Nothing tells you it broke.

A CLAUDE.md grows until the agent quietly stops obeying half of it. A skill's description drifts until it fires on everything, or on nothing. There is no error message and no alert. The agent just gets worse — and the team concludes the model got worse.

Cost fails through accumulation

You paid for that file dump 27 times.

Usage is context length × turns — every turn re-sends the entire conversation. The forty files pasted at turn 3 are billed again at turn 30, and on every turn in between. The team concludes the model got expensive; the session just never got cleaned.

Compliance fails through friction

Nobody regenerates an SBOM mid-sprint.

Not ignorance — tedium. The work is always less urgent than the feature, until it is suddenly a deadline. EU Cyber Resilience Act vulnerability-reporting obligations begin 11 September 2026 for anyone shipping software into the EU.

Model routing · why you hit the limit

You don't pay per prompt. You pay for the whole conversation, every turn.

Every turn re-sends the entire context back to the model — that is how a stateless API carries a conversation at all. A long context isn't paid once; it is paid again on every turn that follows. The model-routing skill teaches that mechanism, then works the levers in order of what they actually save — and "switch to a cheaper model" is third, not first.

Lever 01 · delegate

Read-heavy work goes to a subagent.

A survey of forty files costs the main conversation almost nothing if a subagent reads them and returns only the conclusion — the files live in its context, disposable after the answer. Done inline, they sit in yours and are re-paid on every turn for the rest of the session.

Lever 02 · route

Cheaper models — where there's no judgement to buy.

Most tasks are not architecture. A mechanical, fully-specified edit doesn't need the most capable model available, and a subagent's model: field pins the cheap tier deliberately instead of by accident. The routing table below is the whole decision.

Lever 03 · hygiene

The context you never cleaned.

Pasted command output, whole files read where a grep would have answered, one session dragged across unrelated tasks. /context shows what is silently costing you on every turn; /compact and a fresh session are the fix.

Task shapeTierWhy
Mechanical, fully specified — a rename, boilerplate, transcribing a spec haiku There is no judgement to make. Paying for reasoning you don't use is pure waste.
Integration, multi-file work, debugging, pattern-matching sonnet Needs judgement, but not deep design.
Architecture, design, hard review, subtle correctness opus · fable A wrong architectural call costs more than every token saved by going cheap.

The counter-rule: turn count beats token price. A cheap model that takes three times the turns re-sends the whole context on every extra turn — and ends up costing more than the capable model that got it right first time. "Always use Haiku" is the wrong lesson; least total cost to a correct result is the goal. And /fast is a latency lever, not a cost lever — same model, higher per-token price.

Harness assurance

Scaffolding you can audit. Triggers you can prove.

A skill that never fires is dead code that looks alive. A skill that fires on everything is noise with a good reputation. Both are invisible until someone measures them — harness-audit shows you what's there, harness-eval proves whether it works.

harness-audit

First, see what's there.

Inventories the eight surfaces a repo hands an agent — CLAUDE.md, skills, subagents, commands, hooks and the rest — and reports what's missing, what has drifted, and what's competing for attention. Start here.

harness-eval · the quorum

Three blind judges per prompt.

Every trigger prompt is judged by three independent subagents: no tools, one prompt each, one pinned model. They see only the description: — because that is all the real router sees at trigger time. A judge that can read the skill body is not a judge; it's a cheat.

harness-eval · the verdict

FLAKY is a finding, not a rounding error.

A 2/3 split never rounds up to PASS. A clause the judges can't read consistently is a clause the router won't apply consistently — the disagreement is the result, and the judges' reasoning names the exact phrase to sharpen.

harness-eval · regression

A built-in skeptic marks your homework.

Pin an observable behavior, change one thing in the harness, re-run — then dispatch a skeptic whose only job is to argue the change did nothing. If it can, you haven't shown anything yet. A change that moves nothing is a finding too: that paragraph is dead weight.

This is the same discipline this repo runs on itself — the 372 blind judgements in the evidence section below are harness-eval applied to its own catalogue. What we ship is what we use.

EU compliance evidence

11 September 2026 is not moving.

That is when EU Cyber Resilience Act vulnerability-reporting obligations begin for anyone shipping software into the EU. Compliance fails through friction, not ignorance — so the compliance track does the assembly work continuously and hands humans a review-ready draft, not a deadline panic.

cra-evidence

One skill, one dossier.

SBOM generation, supply-chain diff between releases, vulnerability scans with triage drafts, a CRA Annex I gap report, EAA/WCAG accessibility scans — assembled into a compliance/ directory that is ready for review, not reconstruction.

The contract

Claude drafts. Humans approve.

Every artifact ships as Draft or Proposed — the skill never marks anything Accepted or Compliant on its own initiative. CRA class and EAA scope are legal determinations: asked once, recorded, never assumed. Legal interpretation is deferred to counsel, explicitly.

adr-management

Why is the system like this?

Drafts the architecture decision record at the moment the decision is made, instead of never. And when a vulnerability triage accepts a risk, that decision becomes an ADR too — the two skills hand off to each other.

The hook

A skill is advice. A hook is physics.

Everything in a skills catalogue is a procedure a model should follow — and a model under pressure drops procedures, silently. The gate is the one artifact here it cannot skip.

The rule

Stale, not absent

It blocks a completion claim when a source file changed after the last passing test run. Not “did the tests ever run” — that waves through the exact failure it exists to catch: run the suite, make one more edit, declare victory.

Passed, not merely ran

It knows the difference

Claude Code fires PostToolUse only when a tool call succeeds. So if the recorder ran, the command exited zero. Verified empirically before it was relied on: false | true fires it; with pipefail, it doesn't.

Fails open, on purpose

It will never trap you

Anything it doesn't understand allows the turn. A hook that blocks because it is broken gets uninstalled within a day — and then it enforces nothing, forever. The only way to be blocked is to actually have stale tests.

Evidence, never assertion

We don't ask you to take our word for it.

That's the contract the whole repo is built on, and it applies to the repo itself.

Blind quorum

Every skill's trigger is judged by three strangers

Three independent subagents, no tools, one prompt each. They see only the description: — because that's all the real router sees. A split vote is recorded FLAKY and never rounded up to PASS. 372 judgements. The ledger keeps the rounds that failed.

A fixture built to be broken

The audit is tested against a bad repo

Five deliberate defects it must catch. A tool validated only against a healthy repo proves nothing. Its README says, in as many words: do not fix these — they are the test.

Observed, not asserted

The block fired in a real session

Plugin installed, headless run: the hook fired, the agent read the message, went and ran the tests. Captured from platform debug logs — not synthetic input. 435 automated checks across nine suites, on Linux and macOS.

And what it doesn't guarantee. The Bash-edit detector is a text heuristic, not a shell parser. Timestamps are second-granular. The claim detector is a regex and will miss phrasings — it missed one during review, and that is recorded rather than hidden. The quorum's sample is small, and says so. All of this is on the tin, not in a footnote.

Contribute

The bar is concrete, and it's the interesting part.

Most skill repos accept a markdown file and hope. Here, a skill isn't done until it's proven to fire — and that turns out to be the hard bit.

01

The description is the router

It is the only thing seen at trigger time. A “do NOT use this for X” carve-out in the body cannot prevent a false trigger — the body is invisible when the decision is made.

02

So you must test it

At least five prompts that must fire and five that must not — including a neighbouring skill's territory, because cross-triggering is the failure that actually bites.

03

And every script gets a contract test

bash or python3 stdlib only. No new dependencies. If it degrades, it degrades loudly. CI runs all nine suites on Linux and macOS.
